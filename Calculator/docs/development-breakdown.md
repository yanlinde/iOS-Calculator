# Calculator 开发任务拆解

## 1. 里程碑

### M1：项目初始化
目标：创建可运行的 iOS 工程骨架

任务：
- 创建 SwiftUI iPhone App 工程
- 配置最低系统版本
- 建立目录结构
- 建立主题颜色和基础常量
- 建立测试 Target

交付结果：
- App 可运行
- 空白主页面可启动

### M2：计算引擎
目标：先把最核心、最容易出错的逻辑做稳定

任务：
- 定义 Token 和运算符优先级
- 实现中缀表达式解析
- 实现后缀表达式求值
- 实现 `%` 和 `+/-` 行为
- 实现错误状态处理
- 完成单元测试

交付结果：
- 计算逻辑可独立测试
- 核心表达式用例全部通过

### M3：主页面静态 UI
目标：把设计稿完整还原出来

任务：
- 实现主页面整体布局
- 实现历史区域样式
- 实现双层结果显示区
- 实现 5x4 按键区
- 完成功能键、数字键、运算键的样式区分

交付结果：
- 静态界面与设计稿高度接近

### M4：交互与状态流转
目标：把按钮事件接到显示状态机

任务：
- 接入 ViewModel
- 点击数字和运算符更新输入表达式
- 实时刷新上方预览结果
- 等号后切换显示状态
- 接入删除、清空、百分比、正负切换

交付结果：
- 主流程可完成一整次计算

### M5：历史记录持久化
目标：让产品达到可上线使用状态

任务：
- 建立 SwiftData 模型
- 成功计算后写入历史
- 限制历史最多 10 条
- 历史区滚动展示
- 确保最新一条显示在最下方

交付结果：
- 关闭重开 App 后历史仍在

### M6：质量与上架准备
目标：进入提测与上架材料准备阶段

任务：
- 补齐异常和边界处理
- 完成 UI 测试
- 真机验证主流 iPhone 尺寸
- 准备图标、截图、描述、关键词、隐私信息

交付结果：
- 可打包 TestFlight
- 可进入 App Store Connect 提交流程

## 2. 详细任务列表

## A. 工程初始化
- 创建 Xcode 工程
- 配置 Bundle Identifier
- 配置 App 名称
- 配置 Assets 和颜色资源
- 配置测试 Target
- 建立基础目录结构

## B. 领域层
- 定义 `OperatorType`
- 定义 `Token`
- 实现 `ExpressionTokenizer`
- 实现 `ExpressionParser`
- 实现 `ExpressionEvaluator`
- 实现 `CalculatorFormatter`
- 实现错误枚举

## C. 界面层
- 创建主容器页面
- 创建历史区组件
- 创建小字预览结果组件
- 创建大字主显示组件
- 创建键盘容器组件
- 创建单个按钮组件

## D. 状态管理
- 创建 `CalculatorViewModel`
- 实现输入状态机
- 实现预览结果更新
- 实现等号态切换
- 实现错误态恢复

## E. 数据层
- 创建 `CalculationHistoryItem`
- 创建历史仓储接口
- 创建 SwiftData 实现
- 实现最多 10 条裁剪逻辑
- 实现加载与保存流程

## F. 测试
- 表达式计算测试
- 输入状态切换测试
- 历史记录排序测试
- 历史最多 10 条测试
- UI 布局与交互冒烟测试

## 3. 开发优先级

### P0
- 计算引擎
- 主页面 UI
- 等号逻辑
- 历史记录持久化
- 错误处理

### P1
- 显示格式化优化
- 长文本缩放策略
- UI 测试补齐

### P2
- 图标和商店素材完善
- 埋点和统计能力（如未来需要）

## 4. 预估工期

在单人开发前提下，按稳妥节奏建议：
- 工程初始化：0.5 天
- 计算引擎与测试：1 到 1.5 天
- 静态 UI：1 天
- 交互接通：1 天
- 持久化与历史：0.5 天
- 联调与修复：1 天
- 上架材料整理：0.5 到 1 天

总计：
- `5.5 到 6.5 天`

## 5. 开发验收清单
- 主界面布局与设计稿一致
- 按钮点击反馈正常
- 严格数学优先级正确
- 双层结果区行为正确
- 历史记录持久化正常
- 最多保留 10 条
- 最新一条在最下方
- 历史区域可滚动
- 除零和非法输入不崩溃

## 6. 建议的首个开发顺序
- 先建工程
- 再写计算引擎测试
- 测试通过后再铺 UI
- 最后再接持久化和上架细节

这样可以尽早锁定最关键的正确性，避免后面反复返工。
