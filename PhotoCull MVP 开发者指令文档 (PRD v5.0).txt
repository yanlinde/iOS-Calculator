# PhotoCull MVP 开发者指令文档 (PRD v5.0)

## 1. 项目语境 (Context)
- **产品定义**：极简 iOS 原生体验选片工具。
- **目标场景**：用户旅游、聚会后产生海量照片，需快速从中挑选出几张“朋友圈神图”。
- **技术栈**：React + TypeScript + Tailwind CSS + Vite + Capacitor (用于调用 iOS 相册 API)。
- **核心逻辑**：单阶段、沉浸式筛选。用户在主视图滑动筛选时，可随时对相似照片进行即时对比，无需切换模式。
- **性能目标**：必须支持单次处理 500 张大图，确保 60fps 滑动顺畅，实现内存动态回收。

---

## 2. 数据结构定义 (Data Schema)
> **AI 指令**：请严格使用以下接口定义状态管理，确保筛选逻辑的一致性。

```typescript
type PhotoStatus = 'pending' | 'kept' | 'passed';

interface UserPhoto {
  id: string;            // 随机唯一 ID
  assetId: string;       // iOS 系统相册原始资产标识符
  localUrl: string;      // 本地预览图 Blob URL
  status: PhotoStatus;   // 状态：待定(pending)、保留(kept)、淘汰(passed)
  dimensions: { 
    width: number; 
    height: number; 
  };                     // 用于 UI 布局适配与排序
  creationTime: number;  // EXIF 时间戳，用于按拍摄时间线排序
}

## 3. 功能模块详细需求 (Specifications)
3.1 零延迟导入与预处理
导入上限：单次从 iOS 相册读取最多 500 张照片。

渲染要求 (性能核心)：

禁止一次性解码 500 张原图。必须实现虚拟列表 (Virtual List) 或按需加载机制。

图片进入视口（Viewport）时生成 URL.createObjectURL，离开视口时立即执行 revokeObjectURL 释放内存。

排序逻辑：默认按拍摄时间排序；同时间段内，将横屏照片排在一起，竖屏照片排在一起，方便用户在相似构图中快速扫视对比。

3.2 混合筛选交互 (Hybrid Selection Flow)
单图滑动模式 (Main Stream)：

屏幕中心展示当前照片卡片，背景采用沉浸式全黑。

手势交互：支持左滑/右滑卡片飞出。

左滑或点击红色[X]按钮：照片标记为 passed，卡片向左飞出并加载下一张。

右滑或点击蓝色[心形]按钮：照片标记为 kept，卡片向右飞出并加载下一张。

即时对比功能 (Instant Compare)：

触发方式：在单图界面点击“对比”按钮或向上轻扫卡片。

对比逻辑：App 自动拉取当前照片及其后续第一张 pending 状态的照片进行分屏显示（左右对拼）。

同步缩放 (Sync-Zoom)：双指缩放其中一张图时，另一张图同步缩放倍数与位移中心，方便比对焦点。

决策退出：用户点击其中一张图下方的“选它”，该图状态更新为 kept，另一张自动更新为 passed，随后退出对比返回主流视图。

3.3 社交预览中心 (SNS Hub)
功能：底部常驻已选照片的小缩略图计数器。

预览增强：点击展开 3x3 画布模拟朋友圈预览。支持长按拖拽照片调整顺序，帮助用户判断整组照片的视觉风格是否统一。

## 4. iOS App Store 上架合规与体验
4.1 UI 适配规范
安全区域 (Safe Area)：所有交互组件必须避开 iPhone 顶部的刘海/灵动岛及底部的 Home Indicator 手势条。

点击尺寸：所有操作按钮热区不小于 44x44 pt。

视觉主题：全深色模式。背景：Slate-950，卡片：Slate-900，强调色：Blue-600。

4.2 隐私与权限说明
权限触发：禁止在 App 启动时立即申请相册权限。必须在用户点击“导入”或“开始”按钮后弹出自定义引导界面，再触发系统原生权限请求。

权限描述 (Info.plist)：

NSPhotoLibraryUsageDescription: "我们需要访问您的相册以帮助您快速筛选旅游照片。"

NSPhotoLibraryAddUsageDescription: "我们需要权限将您选中的精选集保存到独立相册中。"

4.3 导出逻辑
功能集成：调用 @capacitor-community/media 插件。

行为规范：在系统相册中自动创建（或查找）名为“层层选片_精选”的文件夹。

操作逻辑：将所有标记为 kept 的照片保存至该文件夹。保存成功后弹出 iOS 原生分享表单或完成提示。

## 5. 开发者特殊提示 (Implementation Notes)
Undo 机制：必须实现一个简易的 Undo 栈（记录最后 10 步操作），允许用户一键撤回误操作。

离线运行：App 必须在完全断网的情况下能够正常运行所有导入、筛选和导出逻辑，禁止依赖任何云端接口。

触感反馈 (Haptics)：

标记为 kept 时调用 impactOccurred('medium')。

标记为 passed 时不调用或调用极轻微反馈。

容错处理：

若单张图片加载失败，应显示占位图并允许跳过，不得阻塞整个筛选流程。

若导入数量巨大，需有显眼的进度条或 Loading 状态提示，防止 UI 假死。
